# SaaS Pilar 2 — Allowlist de archivos con `createAdminClient`.
#
# Cada línea es un path relativo al repo que puede importar/llamar
# `createAdminClient` legítimamente. Cualquier archivo FUERA de esta lista
# que use `createAdminClient` hace fallar `scripts/check-admin-client.sh`.
#
# Formato: `path  # categoría — justificación breve`
# Líneas vacías y comentarios se ignoran.
#
# Categorías:
#   AUTH              — pre-sesión (no hay JWT todavía).
#   BILLING           — operaciones de billing scoped por user.org_id.
#   CRON              — jobs cross-tenant por diseño; insertos llevan org_id explícito.
#   DEFINITION        — archivo que define/exporta `createAdminClient`.
#   IMPORT            — bulk imports con org_id inyectado + RPC p_org_id.
#   LIB               — libs internas fire-and-forget o cross-tenant por diseño.
#   LEGACY_PAGE       — server component legacy; migrar en Pilar 3.
#   PLATFORM_ADMIN    — consola de plataforma, cross-org por diseño; protegido por isPlatformAdmin().
#   RPC_BLOCKED       — routes que llaman `execute_readonly_query` (SECURITY DEFINER); Pilar 2c.
#   STORAGE           — uploads a Supabase Storage; bucket policies son scope separado de DB RLS.
#   TEST_MOCK         — mock de tests, no corre en producción.
#   WEBHOOK_S2S       — webhook server-to-server; org resuelta por token en URL.
#   WRITE_VALIDATED   — writes que validan ownership vía parent RLS antes del admin mutation.
